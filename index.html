What ASeek does

Local file search

Build a local index for selected folders and quickly search file names, folders, and applications without sending your index to a server.

Keyboard-first access

Open the search panel with a global shortcut, type, navigate results, and open files without breaking focus.

Flexible scan roots

Add only the folders you want ASeek to scan. macOS folder permissions are requested explicitly and can be changed later.

Useful previews

Review key file details and preview supported items before opening them, so search results are easier to compare.

Advanced Search

ASeek supports Everything-style search operators for narrowing results without leaving the search box.

ext:pdfLimit results to one extension
path:downloadsNarrow matches to a path fragment
name:reportSearch only within filenames
folder: file:Restrict results to folders or files
!draftExclude unwanted keywords
"annual report"Treat a spaced phrase as one unit

* matches any length, ? matches one character, spaces act as AND, and filters can be combined.

ext:rs path:srcRust files under a src path
invoice !draftInvoices excluding draft matches
path:components !test ext:vueVue components excluding tests
folder: name:src*Folders whose names start with src

FAQ

Does ASeek upload my files or search queries?

No. ASeek is designed for local indexing and local search. It does not upload file names, file paths, file contents, or search queries.

Why do I need to choose folders?

macOS sandboxing requires explicit folder authorization. ASeek indexes only the folders you add.

Why can I not find a file?

Check whether the folder is included in scan roots, whether the file is inside an ignored folder, and whether indexing has completed.

Can I change the global shortcut?

Yes. Open ASeek Settings and record a new shortcut in General settings.
